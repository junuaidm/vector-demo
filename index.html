What Are Vectors?

In the context of AI and machine learning, a vector is a fixed-length array of numbers that represents the meaning of a piece of text. Think of it as coordinates in a high-dimensional “meaning space” — words or sentences with similar meanings land close together.

The interactive plot below shows a simplified 2D projection of 15 words. Notice how semantically related words (vehicles, colors, clothing, gear) naturally form clusters. Real embedding models like nomic-embed-text produce 768-dimensional vectors — far more nuance than we can visualise, but the principle is identical.

Dimensionality matters: This project uses two models — nomic-embed-text (768 dimensions)

Cosine Similarity Explained

Cosine similarity measures the angle between two vectors, not the distance between their endpoints. Two vectors pointing in the same direction have a cosine similarity of 1; perpendicular vectors score 0; opposite vectors score −1.

cos(θ) = (A · B) / (|A| |B|)

Cosine Distance vs. Cosine Similarity

SQL Server’s vector_distance('cosine', ...) returns cosine distance, not similarity. The relationship is:

distance = 1 − similarity     (range: 0 = identical, 2 = opposite)

A distance of 0 means the vectors are identical; closer to 2 means they point in opposite directions. When ranking search results, you sort by distance ascending.

Cosine vs. Euclidean Distance

Cosine cares about direction (what the text means) while ignoring magnitude. Euclidean cares about absolute position in vector space. For text embeddings, cosine is almost always preferred because embedding models normalise direction to encode meaning, while magnitude can vary with text length.

Section 3

Embedding Models

An embedding model is the neural network that converts text into vectors. Different models produce different dimensionalities and capture different nuances. This project uses two models served locally by Ollama:

nomic-embed-text

768 dimensions — general-purpose embedding model. Used for the AdventureWorks product search and country master-data demos.

How It Works

Ollama exposes a REST API at :11434. An NGINX reverse proxy adds TLS (port 443), which SQL Server requires for external model calls. The flow is:

Text  →  HTTPS POST to Ollama via NGINX  →  VECTOR(768)

SQL Server calls this transparently via AI_GENERATE_EMBEDDINGS().

Section 4

SQL Server 2025 Vector Features

SQL Server 2025 introduces native vector support. Here are the key features demonstrated in this project, with actual code from the POC:

1. VECTOR Data Type

Store fixed-length vectors directly in a column:

CREATE TABLE ProductEmbeddings (
    ProductID   INT PRIMARY KEY,
    chunk       NVARCHAR(2000),
    embeddings  VECTOR(768)
);

2. CREATE EXTERNAL MODEL

Register an Ollama embedding endpoint so SQL Server can call it:

CREATE EXTERNAL MODEL ollama
WITH (
    LOCATION   = 'https://model-web:443/api/embed',
    API_FORMAT = 'Ollama',
    MODEL_TYPE = EMBEDDINGS,
    MODEL      = 'nomic-embed-text'
);

3. AI_GENERATE_EMBEDDINGS()

Convert text to a vector in a single function call:

INSERT INTO ProductEmbeddings (ProductID, chunk, embeddings)
SELECT
    p.ProductID,
    p.Name + ' ' + ISNULL(p.Color, '') + ' ' + c.Name AS chunk,
    AI_GENERATE_EMBEDDINGS(
        p.Name + ' ' + ISNULL(p.Color, '') + ' ' + c.Name
        USE MODEL ollama
    ) AS embeddings
FROM [SalesLT].[Product] p
JOIN [SalesLT].[ProductCategory] c
  ON p.ProductCategoryID = c.ProductCategoryID;

4. vector_distance()

Compute cosine distance between two vectors:

DECLARE @search VECTOR(768) =
    AI_GENERATE_EMBEDDINGS('red bike budget' USE MODEL ollama);

SELECT TOP(5)
    pe.ProductID,
    p.Name,
    vector_distance('cosine', @search, pe.embeddings) AS distance
FROM ProductEmbeddings pe
JOIN [SalesLT].[Product] p ON pe.ProductID = p.ProductID
ORDER BY distance;

5. CREATE VECTOR INDEX (DiskANN)

Create an approximate nearest-neighbour index for fast search:

ALTER DATABASE SCOPED CONFIGURATION
    SET PREVIEW_FEATURES = ON;

CREATE VECTOR INDEX vec_idx
ON ProductEmbeddings([embeddings])
WITH (metric = 'cosine', type = 'diskann', maxdop = 8);

6. vector_search() — ANN Queries

Use the index for blazing-fast approximate search:

SELECT TOP(10)
    t.ProductID,
    s.distance
FROM vector_search(
    table      = ProductEmbeddings AS t,
    column     = [embeddings],
    similar_to = @search,
    metric     = 'cosine',
    top_n      = 10
) AS s
ORDER BY s.distance;
Section 5

End-to-End Walkthrough

Demo A: AdventureWorks Product Search

  1. Prepare text chunks — Combine product Name + Color + Category + Model + Description into a single searchable string.
  2. Generate embeddings — Call AI_GENERATE_EMBEDDINGS() on every chunk to produce 768-dimensional vectors.
  3. Create vector index — Build a DiskANN index on the embeddings column for sub-millisecond search.
  4. Run a semantic query — Search for “red mountain bike under $500” and get ranked results by cosine distance.
  5. Interpret results — Lower distance = closer match. Distance 0.0 would be an exact semantic duplicate.

Performance: Full Scan vs. ANN Index

Without an index, every query scans all rows and computes cosine distance on each one. With a DiskANN vector index, the query uses approximate nearest-neighbour search — orders of magnitude faster for large datasets.

Section 6

Architecture

The entire stack runs in Docker Compose with 6 orchestrated services. SQL Server connects to Ollama through an NGINX TLS proxy because CREATE EXTERNAL MODEL requires HTTPS.

Why the TLS Proxy?

SQL Server 2025 requires HTTPS endpoints for external model calls. Since Ollama only serves plain HTTP, an NGINX reverse proxy with a self-signed certificate bridges the gap. The certificate is generated at startup by the config service and shared via a Docker volume.

Services

config

Alpine container that generates self-signed SSL certificates at startup.

ollama

Runs embedding models (nomic-embed-text

model-web (NGINX)

TLS reverse proxy exposing Ollama over HTTPS on port 443.

model-puller

Sidecar that pulls the required models into Ollama on first startup.

sql1

SQL Server 2025 with vector support. Trusts the generated TLS certificate.

sql-tools

Runs configure_model.sql to register the external model endpoint.

Section 7

Getting Started

Prerequisites

  • Docker Desktop (or Docker Engine + Compose)
  • ~8 GB RAM available for the Ollama models
  • SQL Server Management Studio or Azure Data Studio for running the demo SQL files

Quick Start

docker compose up -d

This starts all 6 services. On first run, model download may take a few minutes.

Run the Demos

  1. Connect to localhost:1433 with SA credentials from the compose file.
  2. Run vector-demos.sql — AdventureWorks product search (Demo A).

Each SQL file is self-contained with comments explaining every step.
